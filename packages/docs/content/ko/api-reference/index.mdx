# API 레퍼런스

huni.builder의 모든 API 엔드포인트와 데이터 구조를 문서화합니다.

## API 개요

```mermaid
flowchart TB
    subgraph Client["클라이언트"]
        Widget["huni.builder Widget"]
        Admin["Admin Panel"]
        External["외부 시스템"]
    end

    subgraph API["API Server (NestJS)"]
        AdminAPI["Admin REST API<br/>27 Controllers"]
        WidgetAPI["Widget API<br/>Hono"]
    end

    subgraph Data["데이터 계층"]
        DB[(PostgreSQL<br/>50+ Tables)]
        MES["MES BackOffice"]

    end

    Widget --> WidgetAPI
    Admin --> AdminAPI
    External --> WidgetAPI

    WidgetAPI --> DB
    AdminAPI --> DB

    AdminAPI --> MES
    AdminAPI --> Edicus
```

## 인증 방식

### JWT (JSON Web Token)

Admin API에서 사용합니다.

```
Authorization: Bearer {jwt_token}
```

### API Key

Widget API에서 사용합니다.

```
X-API-Key: {api_key}
```

### HMAC-SHA256

Edicus B2B API에서 사용합니다.

```
X-API-Key: {api_key}
X-Signature: {hmac_signature}
X-Timestamp: {unix_timestamp}
```

## 표준 응답 형식

### 성공 응답

```json
{
  "success": true,
  "data": { /* 실제 데이터 */ },
  "error": null,
  "meta": {
    "timestamp": "2025-12-19T10:00:00Z"
  }
}
```

### 에러 응답

```json
{
  "success": false,
  "data": null,
  "error": {
    "code": "ERR_VAL_001",
    "message": "검증 오류",
    "details": [
      {
        "field": "quantity",
        "message": "수량은 100 이상이어야 합니다"
      }
    ]
  },
  "meta": {
    "timestamp": "2025-12-19T10:00:00Z"
  }
}
```
