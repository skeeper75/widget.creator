# NNGroup 컨텍스트 메뉴 가이드라인 분석

> 출처: "Designing Effective Contextual Menus: 10 Guidelines" - Kate Kaplan, NN/g (2025.11.28)

## 핵심 정의

**컨텍스트 메뉴(Contextual Menu)**: 특정 UI 요소, 인터페이스 영역, 데이터, 뷰에 관련된 액션들의 집합. 주로 부차적인 기능을 담아 필요할 때 접근할 수 있게 함.

## 아이콘 인식 연구 결과

| 아이콘 | 이름 | 인식률 | 사용자 해석 |
|--------|------|--------|------------|
| ⋮ | Kebab | ✅ 높음 | "더 많은 옵션", "추가 기능" |
| ⋯ | Meatball | ✅ 높음 | "다른 액션", "숨겨진 기능" |
| ☰ | Hamburger | ⚠️ 주의 | "메인 네비게이션" (컨텍스트 메뉴 아님) |

## 트레이드오프

### 장점
- 시각적 노이즈 감소
- 레이아웃 간소화
- 집중적 인터랙션 지원

### 단점
- **발견 가능성 저하**: 작거나 희미하면 완전히 놓침
- **낮은 정보 향기**: 메뉴 내용 예측 불가
- **오해 위험**: 캐러셀 컨트롤, 진행 표시기 등과 혼동

## 10가지 가이드라인 상세

### 1. 부차적, 비핵심 액션에만 사용

**원칙**: 자주 사용하는 핵심 기능은 숨기지 말 것

```
✅ 컨텍스트 메뉴에 적합:
   - 복제, 공유, 보관
   - PDF 내보내기
   - 즐겨찾기 추가

❌ 반드시 직접 노출:
   - 저장, 확인, 취소
   - 수정 (자주 사용 시)
   - 삭제 (중요 액션)
```

**나쁜 예시 (AT&T)**: 채팅에서 "End Chat"을 컨텍스트 메뉴에 숨김

---

### 2. 관련 콘텐츠 근처에 배치

**원칙**: 게슈탈트 근접성 - 가까운 요소는 관련된 것으로 인식

```jsx
// ✅ 좋음: 카드 우상단에 배치
<div className="relative">
  <CardContent>...</CardContent>
  <div className="absolute top-2 right-2">
    <ContextMenu ... />
  </div>
</div>

// ❌ 나쁨: 페이지 하단이나 멀리 배치
```

**나쁜 예시 (Ramp)**: 폼 필드 우하단 구석에 작은 케밥

---

### 3. 충분한 크기와 대비 확보

**원칙**: 너무 미묘하면 발견되지 않음

```jsx
// ✅ 좋음
className="w-8 h-8 border border-gray-200"
<Icon className="w-5 h-5 text-gray-600" />

// ❌ 나쁨
className="w-4 h-4"
<Icon className="w-3 h-3 text-gray-300" />
```

**좋은 예시 (Rippling)**: 미트볼에 테두리 추가 → 탭 가능성 힌트

---

### 4. 관련 액션끼리만 그룹화

**원칙**: 논리적으로 같은 객체에 속하는 액션만 묶기

```jsx
// ✅ 좋음: 견적 항목 관련만
[수정, 복제, 삭제] + [내보내기, 공유]

// ❌ 나쁨: 전역+항목 혼합
[삭제, 설정, 로그아웃, 도움말]
```

**좋은 예시 (Slack)**: 메시지 메뉴에 메시지 관련 액션만

---

### 5. 인터페이스 전체에서 일관된 동작

**원칙**: 같은 아이콘 = 같은 기능

```
✅ 모든 케밥 → 드롭다운 메뉴
❌ 케밥 A → 드롭다운, 케밥 B → 모달, 케밥 C → 사이드패널
```

**나쁜 예시 (Google)**: 광고 케밥 → 모달, 일반 결과 케밥 → 사이드패널

---

### 6. 툴팁으로 모호함 해소

**원칙**: 케밥/미트볼은 낮은 정보 향기 → 힌트 필요

```jsx
// ✅ 좋음
title="견적 옵션: 수정, 복제, 삭제"

// ❌ 나쁨
title="옵션"
title="Ellipses"  // 리터럴 라벨
```

**좋은 예시 (Notion)**: "Style, Export, and more…"

---

### 7. 액션 전용, 콘텐츠 확장용 아님

**원칙**: 케밥/미트볼 = 액션 메뉴 전용

```jsx
// ✅ 액션용
<MoreVertical /> → 드롭다운 메뉴

// ✅ 콘텐츠 확장용 (별도 패턴)
<button>더 보기</button>
<ChevronDown />
```

**나쁜 예시 (Etsy)**: 리뷰 확장에 미트볼 사용

---

### 8. 단일/극소수 액션에는 사용 금지

**원칙**: 1-2개 액션은 직접 노출이 효율적

```jsx
// ❌ 나쁨
<ContextMenu items={[{ label: '삭제' }]} />

// ✅ 좋음
<button><Trash className="w-4 h-4" /></button>
```

**나쁜 예시 (Weather.com)**: 케밥 → "Delete" 단일 버튼

---

### 9. 햄버거 ≠ 컨텍스트 메뉴

**원칙**: 아이콘 역할 분리 유지

| 아이콘 | 용도 |
|--------|------|
| ☰ Hamburger | 전역 네비게이션, 사이트 메뉴 |
| ⋮⋯ Kebab/Meatball | 항목별 컨텍스트 액션 |

**나쁜 예시 (Banana Republic)**: 채팅에서 햄버거 → "Save Transcript"

---

### 10. 키보드/스크린 리더 접근성

**원칙**: 모든 사용자가 접근 가능해야 함

```jsx
<button
  aria-label="견적 옵션 메뉴"
  aria-haspopup="menu"
  aria-expanded={isOpen}
  onKeyDown={(e) => {
    if (e.key === 'Enter') setIsOpen(true);
    if (e.key === 'Escape') setIsOpen(false);
  }}
>
```
