# 샵 확장 전략 가이드

*스파미 Shopify 오프라인 강의 기반 (2025년 12월)*

> 하나에 매몰 X, 옆으로 확장 O
> 5개 → 10개 → 30개

---

## 1. 핵심 전략: 옆으로 확장

### 왜 확장인가?

```
[잘못된 접근]
1개 샵에서 매출 $10,000 만들기
→ 경쟁, 광고비, 시간 필요
→ 어려움

[올바른 접근]
5개 샵에서 각각 $2,000 만들기
→ 합계 $10,000
→ 훨씬 쉬움
```

### 확장의 장점

| 장점 | 설명 |
|------|------|
| **리스크 분산** | 1개 망해도 나머지 있음 |
| **테스트 가능** | 다양한 카테고리 시도 |
| **병렬 성장** | 5개가 동시에 성장 |
| **비용 효율** | 추가 비용 = 플랜비만 |

---

## 2. 목표 샵 개수

### 단계별 로드맵

```
[1단계] 미니멈 5개
- 입문자 목표
- 1인 관리 가능
- 월 $2,000 × 5 = $10,000

[2단계] 확장 10개
- 내년 목표
- 루틴 잡힌 후
- 월 $2,000 × 10 = $20,000

[3단계] 2인 운영 30개
- 가족/파트너와 함께
- 월 $2,000 × 30 = $60,000

[4단계] 전업 50개
- 가장 많이 운영하는 수강생 수준
- 완전 올인
```

### 왜 1인이 10개 가능한가?

```
샵 수 ≠ 일의 양
주문 건수 = 일의 양

예:
- 10개 샵 × 일 2건 = 일 20건 처리
- 발주 + CS = 하루 1-2시간

→ 샵 늘어도 일 선형 증가
→ 1×1이 아니라 N×상수
```

## 3. 비용 구조

### 샵 추가 비용

```
[샵당 월 비용]
- Shopify Basic 플랜: $39
- 그 외 필수 비용: 없음

5개 샵 = $195/월
10개 샵 = $390/월
```

### 비용 대비 수익

```
[손익분기점]
1개 샵에서 월 1건만 팔아도:
- 마진 $20 × 1건 = $20
- 플랜비 $39
- 손실 $19

2건만 팔면:
- 마진 $20 × 2건 = $40
- 플랜비 $39
- 이익 $1

→ 월 2건만 팔면 흑자
→ 5만개 상품 중 2개만 팔리면 됨
```

## 4. 매몰 금지 원칙

### 잘못된 접근

```
❌ "이 샵 매출이 안 나와... 
    어떻게 살릴까? 뭘 바꿔볼까?"

→ 시간 낭비
→ 정답 없음
→ 기회비용
```

### 올바른 접근

```
✅ "이 샵 안 나오네. 냅두고
    새 샵 하나 더 만들자"

→ 새 샵 = 새 기회
→ 기존 샵도 언젠가 터질 수 있음
→ 병렬 실험
```

### 시간 사용 원칙

```
남는 시간이 있다면:
- 기존 샵 수정/개선 → X
- 새 샵 생성 → O

"샵 수정할 시간에 샵 만들어라"
```

## 5. 문제 샵 처리

### 차지백 누적 샵

```
[상황]
- 사기꾼들 사이에 "호구 샵" 소문
- 정상 고객 < 사기 주문
- 차지백 비율 1% 넘음
- 노출 제한됨

[해결]
- 해당 샵 폐쇄
- 새 샵으로 갈아타기
```

### 폐쇄 방법

```
1. Shopify Admin → Settings
2. Plan → Deactivate store
3. 완전 폐쇄 또는 일시정지

→ 플랜비 중단
→ 데이터 보존 기간 있음
```

---

## 6. 예비 샵 전략

### 왜 예비 샵?

```
문제 발생 시 즉시 전환용:
- 차지백 누적 샵 폐쇄 시
- IP 신고 반복 시
- 기타 문제 시
```

### 예비 샵 설정

```
[권장]
- 운영 샵 + 예비 2-3개 생성
- 기본 설정만 완료
- 비밀번호(Lock) 걸어두기
- 최소 플랜 유지

[비용]
Pause and Build 플랜: $9/월
→ 예비 3개 = $27/월
```

### Lock 설정

```
Settings → Password protection → Enable

→ 고객 접근 불가
→ 노출 안 됨
→ 필요할 때 열기
```

## 7. 업무 위임

### 감당 안 될 때

```
샵 30개 이상:
- 발주 + CS 시간 증가
- 혼자 힘들 수 있음

해결책:
1. 파트타임 채용
2. 대행 서비스 이용
```

### 대행 서비스

```
GSK 등에서 제공:
- 발주 대행
- CS 대행
- 블로그 포스팅 대행

→ 비용 대비 시간 절약
→ 샵 확장에 집중 가능
```

### 파트타임 활용

```
발주 + CS = 단순 반복
→ 매뉴얼화 가능
→ 파트타임으로 충분

채용 포인트:
- 영어 기본 가능
- 꼼꼼함
- 시간 엄수
```

## 8. 확장 타이밍

### 언제 확장?

```
[조건]
1. 기존 샵 기본 설정 완료
2. 상품 5만개 등록 완료
3. 쇼피파이 페이먼트 심사 중
4. 블로그 포스팅 시작
5. 첫 주문 경험

→ 위 완료 후 다음 샵 시작
```

### 확장 사이클

```
Week 1-2: 샵 A 설정
Week 3-4: 샵 A 상품 등록
Week 5: 샵 A 심사 대기 + 샵 B 시작
Week 6-7: 샵 B 설정
...

→ 파이프라인 방식
→ 동시에 여러 샵 다른 단계
```

## 9. 카테고리 다양화

### 왜 다양화?

```
같은 카테고리 5개 샵:
- 내가 나와 경쟁
- 시장 변동 시 전체 타격

다른 카테고리 5개 샵:
- 리스크 분산
- 트렌드 변화 대응
- 다양한 고객층
```

### 카테고리 조합 예시

```
[포트폴리오 예시]

샵 1: 텀블러/드링크웨어
샵 2: 파티용품
샵 3: 주방용품
샵 4: 가든/아웃도어
샵 5: 사무용품

→ 시즌성 분산
→ 리스크 분산
```

## 10. 확장 시 주의사항

### 계란 바구니

```
Google 계정 분리:
- 샵 2-3개당 1개 Google 계정
- AUP 정책 대비
- 하나 문제되도 나머지 안전
```

### 결제 정보 관리

```
각 샵별 관리:
- 결제 카드
- 정산 계좌
- 연락처

→ 혼동 방지
→ 정산 명확화
```

### 품질 유지

```
확장해도 기본 유지:
- 홈페이지 신뢰도
- 정책 페이지 완비
- CS 응답 시간
- 블로그 포스팅
```

## 11. 사업 정리

### 정리 시

```
샵 폐쇄 시:
1. 미처리 주문 확인
2. 진행 중 CS 완료
3. 정산 확인
4. 플랜 해지

→ 단순한 과정
→ 재고 없으니 처리할 것 없음
```

### 일시 중단

```
본업 바빠서 잠시 쉬고 싶을 때:
1. 비밀번호(Lock) 설정
2. Pause and Build 플랜 전환 ($9)
3. 나중에 다시 열기

→ 완전 폐쇄 아님
→ 데이터 보존
```
