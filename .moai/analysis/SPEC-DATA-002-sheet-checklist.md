## Part A: 색상 코딩 시스템

엑셀 전체에 걸쳐 일관되게 적용되는 색상 코딩 규칙이다. 모든 시트 분석 시 이 규칙을 기준으로 해석한다.

### 컬럼 헤더 색상 (셀 배경색)

| 색상 | HEX 코드 | 의미 | UI 노출 여부 | DB 처리 방침 |
|------|----------|------|-------------|-------------|
| 갈색 (Brown) | `FFC4BD97` | 내부 ID / 식별자 컬럼 | 미노출 | DB 저장, API 미노출 |
| 빨강 (Red) | `FFE06666` | 필수 UI 입력 항목 | **필수 노출** | `NOT NULL` 제약 |
| 오렌지 (Orange) | `FFF6B26B` | 선택 UI 입력 항목 | 조건부 노출 | `NULLABLE` 허용 |
| 회색 (Gray) | `FFD9D9D9` / `FFCCCCCC` / `FFBFBFBF` | 내부 생산 정보 | 미노출 | DB 저장, API 미노출 |
| 녹색 (Green) | `FF92D050` / `FF93C47D` | 디자인 에셋 필드 | 미노출 | 파일명 약어 등 참조용 |
| 분홍 (Pink) | `FFF4CCCC` | Edicus 편집기 전용 | Edicus UI만 | Edicus API 연동 |
| 복숭아 (Peach) | `FFFABF8F` | 가격 기준값 | 미노출 (내부 참조) | 기준 판매가 저장 |

### 행 배경색 (데이터 행 수준)

| 색상 | 의미 | DB 처리 방침 |
|------|------|-------------|
| 노란색 배경 | 신규 상품 (MES 미등록) | `products.mes_status = 'PENDING'` |
| 회색 배경 | 품절 또는 준비중 상품 | `products.status = 'OUT_OF_STOCK'` 또는 `'COMING_SOON'` |

### 글자색 규칙

| 글자 색상 | 의미 | DB 처리 방침 |
|-----------|------|-------------|
| 회색 글자 | 사용자 미노출 (파일사양 구분용) | 내부 참조만, UI 미노출 |
| *오렌지 글자 | 작성자 개인 메모 | **개발 시 무시** |
| ★ 표시 (상품명) | MES 미등록 신규 상품 | `products.mes_status = 'PENDING'` |

### 가격표 시트 추가 색상

| 색상 | HEX 코드 | 의미 |
|------|----------|------|
| 연보라 | `FFD9D2E9` | 계산된 가격 (수식 결과) |
| 라일락 | `FFB4A7D6` | 파생 가격 (기본공급수량 적용) |
| 노란베이지 | `FFFEF1CC` | 일반 동판비 / 다른 가격 구간 |
| 노란색 | `FFFFFF00` | 메인 출력 방식 헤더 |

### Sheet 1: MAP

> 1042행 x 26열 | 전체 상품 카탈로그의 "진실의 단일 출처"

#### 1. 명시적 구조

- **12개 카테고리 열 (A-L)**:
  - `01 엽서` / `02 스티커` / `03 인쇄홍보물` / `04 포스터` / `05 사인` / `06 책자` / `07 캘린더` / `08 문구` / `09 아크릴` / `10 라이프` / `11 에코백` / `12 포장`
- **상단 색상 구분 (3개 대분류)**:
  - 파란색(`FFDAEEF3`) = 인쇄상품군 (01-07)
  - 오렌지색(`FFFBD4B4`) = 굿즈/악세사리군 (08-11)
  - 밝은파란색(`FF00B0F0`) = 포장군 (12)
- **카테고리별 상품 수**: 01엽서(19) / 02스티커(18) / 03인쇄홍보물(22) / 04포스터(22) / 05사인(19) / 06책자(9) / 07캘린더(11) / 08문구(23) / 09아크릴(25) / 10라이프(47) / 11에코백(67) / 12포장(22)
- **신규 상품 현황 (★ 표시, DB 미등록)**:
  - 01엽서: 5개 / 02스티커: 3개 / 04포스터: 7개 / 05사인: 7개 / 09아크릴: 9개 / 10라이프: 10개 / 11에코백: 14개
  - **총 60개 이상 ★ 상품**

#### 2. 암묵지 분석

| 항목 | 숨겨진 의도 | 개발 영향도 |
|------|-----------|-----------|
| ★ 표시 | MES 미등록 신규상품 (DB에도 없음) | 높음 - 신규 등록 파이프라인 필요 |
| 카테고리 색상 3분류 | UI/UX 그룹핑 힌트 (인쇄/굿즈/포장) | 중간 - 카테고리 계층 구조 반영 |
| MAP = 진실의 단일 출처 | 다른 시트에 없어도 MAP에 있으면 실제 상품 | 높음 - MAP 기준으로 상품 존재 검증 |
| Row 순서 | 서비스 표시 우선순위 (숫자 낮을수록 메인) | 중간 - `display_order` 필드 필요 |
| 열(카테고리)별 상품 나열 | 가로 방향 = 카테고리, 세로 방향 = 상품 목록 | 낮음 - 파싱 시 주의 |

#### 3. DB 반영 검토 항목

- [ ] `products` 테이블의 `mes_status` 필드: `PENDING`(★) / `REGISTERED` / `NOT_REQUIRED` ENUM 정의
- [ ] `products` 테이블의 `category_id`: MAP 카테고리 12개 매핑 확인
- [ ] 카테고리 대분류(인쇄/굿즈/포장) 3가지 → `product_categories` 테이블에 계층구조(`parent_id`) 반영 여부
- [ ] ★ 상품 60개의 신규 등록 계획 수립 (개발 우선순위 결정)
- [ ] `products.display_order` 필드: MAP Row 순서 반영
- [ ] 각 카테고리별 상품 수 DB와 MAP 일치 여부 교차 검증

#### 4. 미확인 사항

- [ ] MAP에만 있고 상세 시트에 없는 상품은 어떻게 처리하는가? (예: MAP에 이름만 있고 옵션 정의가 없는 경우)
- [ ] 카테고리 열 순서(Row 순서) = 실제 UI 표시 순서인가, 아니면 별도 정렬 기준이 있는가?
- [ ] 12개 카테고리 외에 향후 추가 카테고리 계획이 있는가?

### Sheet 2: !디지털인쇄용지

> 1006행 x 31열 | 전체 용지 마스터 데이터 + 상품군별 사용 가능 플래그

#### 1. 명시적 구조

- **A열**: 구분 (디지털인쇄 / 스티커용지 / 색지 / PET / 합판 / T3절) -- 총 6가지 용지 분류
- **B열**: 종이명 (예: 모조100, 아트200 등)
- **C열**: 파일명약어 (Green `FF93C47D` = 디자인 에셋)
- **E열**: 비고
- **F열**: 전지 규격
- **G열**: 연당가 (원/연 단위) -- 핵심 원가 데이터
- **H열**: 가격(국4절) = `G / 2000` 수식 (반올림)
- **I열**: 현재(A4) = `H / 2` 수식
- **J~AD열**: 상품군별 플래그 컬럼 (`#디지털인쇄용지`, `#봉투`, `#엽서`, `#상품권`, `#반칼스티커`, `#완칼스티커`, `#타투스티커`, `#합판스티커` 등)

#### 2. 암묵지 분석

| 항목 | 숨겨진 의도 | 개발 영향도 |
|------|-----------|-----------|
| 연당가(G열)만 관리 | 나머지 가격은 수식으로 자동계산 (`G/2000`, `G/2000/2`) | 높음 - DB에서 계산 vs 저장 결정 |
| `#` 접두사 컬럼 | 해당 상품군에서 이 종이 사용 가능 여부 플래그 | 높음 - 중간 테이블 설계 |
| C열 녹색 파일명약어 | 파일명에 약어가 그대로 들어감 (AI 파일명 규칙) | 중간 - MES 연동 시 사용 |
| 구분 6분류 | 생산공정에 따른 용지 분류 (인쇄 장비별) | 중간 - `paper_category` ENUM |
| T3절 구분 | 특수 사이즈 절단 방식 (전지 기준 다름) | 중간 - 가격 계산식 분기 |

#### 3. DB 반영 검토 항목

- [ ] `paper_types` 테이블: `per_ream_price`(연당가) 컬럼 존재 확인 (`NOT NULL`)
- [ ] `paper_types.quarter_sheet_price` = `per_ream_price / 2000` -- DB에서 실시간 계산? 또는 저장 후 트리거 업데이트?
- [ ] `paper_types.a4_price` = `quarter_sheet_price / 2`
- [ ] 상품군별 종이 플래그 → `product_paper_compatibility` 중간 테이블 (product_category_id, paper_type_id) 설계
- [ ] C열 파일명약어 → `paper_types.file_name_abbr` 필드 저장
- [ ] 구분 6분류 → `paper_types.paper_category` ENUM (`DIGITAL_PRINT`, `STICKER`, `COLOR`, `PET`, `COMBINED`, `T3_CUT`)
- [ ] 전지(F열) 규격 → `paper_types.full_sheet_size` 필드

#### 4. 미확인 사항

- [ ] 국4절 = 연당가 / 2000 공식이 **모든** 용지 종류에 동일하게 적용되는가? (전지 크기 차이로 인한 예외?)
- [ ] T3절 구분의 연당가 계산식이 일반 용지와 다른가?
- [ ] PET/매트지 등 특수 소재도 이 시트에서 관리하는가, 별도 시트가 있는가?
- [ ] `#` 플래그 컬럼의 값이 빈 셀 = 미사용인가, 아니면 특정 값(O/X, Y/N)으로 표시되는가?

### Sheet 3: 디지털인쇄

> 1027행 x 43열 | 엽서, 포토카드, 접지카드, 상품권, 명함, 배경지, 인쇄홍보물

#### 1. 명시적 구조

- **Brown 컬럼**: A(구분), B(ID), C(MES_ITEM_CD), D(상품명)
- **Red 컬럼 (필수)**: E(사이즈), N(주문방법), P(종이), Z(제작수량)
- **Gray 컬럼 (내부)**: G(파일사양), H~M(파일사양 하위 컬럼)
- **Orange 컬럼 (선택)**: Q(인쇄), R(별색인쇄), W(코팅), X(커팅), Y(접지), AD(후가공), AI(박/형압), AL(추가상품)
- **구분(A열) 값**: 엽서, 포토카드, 접지카드, 상품권, 명함, 배경지, 인쇄홍보물

#### 2. 암묵지 분석

| 항목 | 숨겨진 의도 | 개발 영향도 |
|------|-----------|-----------|
| E~M열 다중 컬럼 | 사이즈 = 텍스트 + 가로 + 세로 + 재단/작업사이즈 등 복수 필드 | 높음 - 사이즈 모델 설계 |
| G열 파일사양(gray) | MES 발주에 필요하지만 UI 미표시 정보 | 중간 - 내부 전용 필드 |
| N열 주문방법 | '업로드'(파일업로드) vs 'Y'(편집기사용) → 주문 플로우 결정 | 높음 - 라우팅 로직 |
| 종이(P열) | `!디지털인쇄용지` 시트와 종이명으로 연동 | 높음 - FK 관계 |
| 후가공(AD열) | 가격표 `후가공` 시트와 연동 | 높음 - 가격 합산 로직 |
| 박(AI열) | 가격표 `후가공_박` 시트와 연동 | 높음 - 가격 합산 로직 |
| B열 ID vs C열 MES_ITEM_CD | 내부 순번 vs MES 시스템 코드 (서로 다른 식별 체계) | 중간 - 이중 식별자 관리 |

#### 3. DB 반영 검토 항목

- [ ] `products.order_method` ENUM: `UPLOAD` / `EDITOR` 정의 (N열 기반)
- [ ] `products.category` = 구분(A열) 7가지 매핑 (엽서, 포토카드, 접지카드, 상품권, 명함, 배경지, 인쇄홍보물)
- [ ] `product_options` 테이블: 인쇄/코팅/커팅/접지/후가공/박 각각 별도 옵션 그룹으로 등록
- [ ] 파일사양(G~M열) gray 컬럼 전체 → `product_specs` 또는 `production_specs` 테이블에 저장

| 번호 | 규칙 | DB 처리 |

| 5 | 빨강 헤더 = 모든 상품에 필수 | `NOT NULL` 제약 |

## 부록: 가격 계산 엔진 요약

### 상품 유형별 가격 계산 공식

```
[디지털인쇄 (엽서/명함/홍보물 등)]
최종가 = 용지비 + 출력비 + 후가공비(선택) + 박비(선택)
  용지비 = (연당가 / 2000) x 매수 / 판걸이수
  출력비 = 출력단가(수량별) x 매수
  후가공비 = 후가공단가(수량별) x 매수
  박비 = 동판비(1회) + 박단가(수량별) x 매수

[스티커]
최종가 = 용지비 + 출력비 + 가공비(커팅)
  (3개 시트 합산 -- 디지털용지 + 디지털출력비 + 스티커가격표)

[캘린더]
최종가 = 출력비(국4절 단면수량별) + 용지비(국4절 환산+로스량) + 제본비(가공+부자재)
  일부 캘린더: 상품단가(고정)

[실사/포스터]
최종가 = 기본가(가로x세로 매트릭스) + 코팅비(선택) + 공임비(선택)

[아크릴]
최종가 = 기본가(가로x세로x수량 매트릭스) x 수량할인

[굿즈]
최종가 = 고정 단가

[포토북/디자인캘린더]
최종가 = Edicus API 가격

[명함]
최종가 = 명함 고정가(유형별) + 박비(선택)

[파우치]
최종가 = 조정가(기준가 기반) x (1 - 수량할인율)

[상품악세사리]
최종가 = 고정 단가 x 수량
```
